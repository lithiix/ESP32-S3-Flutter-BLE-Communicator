# ⚡ ASO QUICK REFERENCE GUIDE

## 🚀 QUICK START (5 Minutes)

Your app is ready for ASO! Here's what to do RIGHT NOW:

### 1. Create App Icon

- **Size**: 512x512px
- **Style**: Tech/Blue theme with Bluetooth symbol
- **Tools**: Canva, Figma, or hire on Fiverr ($10-30)

### 2. Take Screenshots (5 images)

- **Dimensions**: 1440x2560px (Android) OR 1242x2208px (iOS)
- **Show**: Device selection, LED control, vehicle mode, voice commands, connection status
- **Tools**: Use emulator or real device's screenshot tool

### 3. Write Privacy Policy

- **Where**: https://www.privacypolicytemplate.com/ (use free template)
- **Include**: Bluetooth usage, microphone, location, analytics, ads
- **Host**: GitHub Pages or privacy-preserving hosting

### 4. Choose One Store to Start

- **Google Play** (Android): Easier, $25 one-time fee
- **App Store** (iOS): More rigorous, $99/year
- → Start with **Google Play** if budget-limited

## 🛠️ BUILD COMMANDS

### Build for Google Play:

```bash
cd flutter_app
flutter clean
flutter pub get
flutter build appbundle --release
# Output: build/app/outputs/bundle/release/app-release.aab
```

### Build for App Store:

```bash
cd flutter_app
flutter clean
flutter pub get
flutter build ios --release --codesign
# Then upload via Xcode or Transporter
```

## ❓ FAQ

**Q: Should I launch on both stores simultaneously?**
A: No, start with Google Play (faster review), then App Store.

**Q: How long does review take?**
A: Google Play: 2-3 hours. App Store: 24 hours typical.

**Q: Will ads hurt my rating?**
A: Shouldn't if implemented properly. Be transparent in description.

**Q: Should I launch as free or paid?**
A: Free + ads is recommended for first launch. Build user base first.

**Q: How often should I update?**
A: Monthly-quarterly basis shows active development.

**Q: What if my app is rejected?**
A: Read rejection reason carefully, fix issue, resubmit (usually accepted within hours).
